# AGENTS.md — AI Coding Agent Instructions

## Project Overview

This is an enterprise-grade Chrome extension starter built with **WXT + Vue 3 + TypeScript + Pinia + Element Plus**. The project follows a **multi-entry SPA architecture** where `popup`, `options`, and `sidepanel` are fully independent Vue applications.

## Technology Stack

- **WXT** `^0.20.x` — Browser extension framework (HMR, auto-reload, cross-browser)
- **Vue 3** `^3.5.x` — Composition API with `<script setup lang="ts">`
- **TypeScript** `^5.9.x` — Strict mode, full type coverage
- **Pinia** `^3.x` — State management, one instance per entry point
- **Vue Router 4** `^4.x` — Hash mode, one router per entry point
- **Element Plus** `^2.x` — UI component library with auto-import
- **@wxt-dev/i18n** `^0.2.x` — WXT native i18n, follows browser locale
- **SCSS** — Global variables injected via `wxt.config.ts`

## Critical Architecture Rules

### 1. Every UI entry has its OWN Vue app, Router, and Pinia instance

```
entrypoints/popup/main.ts   → createExtensionApp(App, { router }) → mount('#app')
entrypoints/options/main.ts  → createExtensionApp(App, { router }) → mount('#app')
entrypoints/sidepanel/main.ts → createExtensionApp(App, { router }) → mount('#app')
```

- **NEVER** share a Pinia instance across entries.
- **NEVER** share a Router instance across entries.
- Each entry calls `createExtensionApp()` which internally calls `createPinia()`.
- Shared state between entries goes through **WXT Storage**, not through shared Pinia.

### 2. i18n is WXT-native, NOT vue-i18n

- Translations live in `locales/en.json` and `locales/zh-CN.json`.
- Import via: `import { i18n } from '#i18n'` (auto-generated by `wxt prepare`).
- Use in templates: `{{ i18n.t('popup.increment') }}`.
- The `lib/i18n.ts` wrapper provides helpers like `getBrowserLocale()` and `isChineseLocale()`.
- **DO NOT** add runtime language switching — WXT i18n follows `browser.i18n.getUILanguage()`.
- **DO NOT** install or use `vue-i18n`.

### 3. State sync uses wxt/storage, not direct cross-context access

- All shared state is defined in `utils/storage.ts` using `storage.defineItem<T>()`.
- The Pinia store (`stores/app.ts`) watches storage changes via `appSettingsStorage.watch()`.
- Background service worker (`entrypoints/background.ts`) handles typed messages.
- When a UI entry needs to read/write shared state, use the Pinia store actions (they persist automatically).

### 4. Content scripts use Shadow DOM isolation

- `entrypoints/content.ts` uses `createShadowRootUi` from `wxt/client`.
- The Vue component mounts inside a Shadow DOM wrapper — styles don't leak.
- Communication with background uses typed messages from `utils/messaging.ts`.

## File Conventions

| Directory | Purpose | Rules |
|-----------|---------|-------|
| `entrypoints/` | WXT entry points | One folder per entry; each with `main.ts`, `App.vue`, `router.ts` |
| `components/` | Shared Vue components | Reusable across entries |
| `composables/` | Vue composables | `useXxx` naming convention |
| `stores/` | Pinia store definitions | Composables-style (`defineStore('name', () => { ... })`) |
| `utils/` | Pure utility functions | Framework-agnostic helpers |
| `lib/` | Core library wrappers | i18n, Element Plus setup, app factory |
| `locales/` | i18n JSON files | Flat key structure with dot notation (`common.theme`) |
| `assets/styles/` | Global SCSS | `variables.scss` auto-injected; includes Element Plus theme overrides |

## TypeScript & Import Rules

- Use `@` alias for project root imports: `import { useAppStore } from '@/stores/app'`.
- Element Plus components and APIs are **auto-imported** — do NOT add manual imports.
- Element Plus icons must be imported explicitly: `import { Close } from '@element-plus/icons-vue'`.
- WXT i18n types are generated by `wxt prepare` — run after changing `locales/*.json`.
- `tsconfig.json` extends `./.wxt/tsconfig.json` with `"element-plus/global"` types.

## Common Patterns

### Creating a new UI entry

```ts
// entrypoints/<name>/main.ts
import { createExtensionApp } from '@/lib/create-extension-app'
import { useAppStore } from '@/stores/app'
import App from './App.vue'
import router from './router'
import '@/assets/styles/global.scss'

async function bootstrap() {
  const { app, pinia } = createExtensionApp(App, { router })
  const appStore = useAppStore(pinia)
  await appStore.init()
  await router.isReady()
  app.mount('#app')
}

void bootstrap()
```

### Using the app store

```ts
import { useAppStore } from '@/stores/app'

// In setup:
const appStore = useAppStore()
await appStore.init() // Initialize from storage
appStore.incrementCounter() // Persists to wxt/storage
appStore.setTheme('dark') // Persists to wxt/storage
```

### Sending typed messages to background

```ts
import { sendExtensionMessage } from '@/utils/messaging'

const result = await sendExtensionMessage({ type: 'incrementCounter' })
```

### Dark mode composable

```ts
import { useDarkMode } from '@/composables/useDarkMode'

const { isDark, theme, setTheme, toggleDark } = useDarkMode()
```

- ESLint uses `@antfu/eslint-config` (flat config in `eslint.config.mjs`).
- Run `pnpm lint:fix` before committing.
- Commit messages follow conventional commits via `cz-git`.

## Things to Avoid

- **DO NOT** add `vue-i18n` or any alternative i18n library.
- **DO NOT** share Pinia or Router instances across entry points.
- **DO NOT** use `window` or `document` in background service worker (it has no DOM).
- **DO NOT** use `localStorage` or `sessionStorage` — use `wxt/storage` instead.
- **DO NOT** manually import Element Plus components (they auto-import).
- **DO NOT** add runtime language switching logic in the UI.
